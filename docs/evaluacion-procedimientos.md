# Evaluación de procedimientos, pruebas y funcionamiento

## Procedimiento de evaluación de actividades
- Se han definido pruebas unitarias para validar la lógica y los tipos de datos.
- Se han realizado pruebas de integración para verificar el funcionamiento completo de los módulos principales.
- Se documenta cada prueba con entradas y salidas esperadas.

## Indicadores de calidad
- Cobertura de pruebas superior al 80% en los módulos críticos.
- Validación de datos (DNI, edad, email) en todos los formularios.
- Ausencia de errores críticos en la ejecución y despliegue.
- Experiencia de usuario positiva (fluidez, mensajes claros).

## Gestión y registro de incidencias
- Las incidencias detectadas durante el desarrollo y pruebas se registran en un documento compartido.
- Se documenta la solución aplicada y el impacto en el sistema.
- Se actualizan los manuales y guías si hay cambios relevantes.

## Gestión de cambios y recursos
- Los cambios en requisitos o recursos se gestionan mediante control de versiones (Git).
- Se registra cada modificación relevante en el historial del proyecto.
- Se informa a los usuarios/clientes de los cambios importantes.

## Participación de usuarios/clientes
- Se han realizado pruebas de usuario con perfiles reales y simulados.
- Se recoge feedback y se ajusta la aplicación según sugerencias.
- Se elaboran documentos específicos para la evaluación de usuarios.

## Cumplimiento del pliego de condiciones
- El sistema cumple con todos los requisitos funcionales y técnicos definidos en la tarea.
- Se ha establecido un sistema de revisión y validación antes de la entrega final.

# Fase de pruebas

## Pruebas unitarias
- Validación de tipos y formatos (edad, DNI, email, etc.).
- Ejemplo: DNI con letra de control, edad positiva, email válido.
- Entradas y salidas esperadas documentadas.

## Pruebas de integración
- Registro, login, puja, notificaciones y eliminación de usuario.
- Escenarios completos con datos reales y simulados.
- Resultados y evidencias (capturas, logs).

## Ejecución
- Todas las pruebas se ejecutan con `dotnet test` y resultados visibles en la terminal.
- Cobertura suficiente para los casos de uso principales.

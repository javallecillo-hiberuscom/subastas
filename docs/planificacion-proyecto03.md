# Planificación y ejecución del Proyecto 03

## Secuenciación de actividades
1. Análisis de requisitos y diseño
2. Configuración del entorno y recursos
3. Desarrollo del backend y frontend
4. Implementación de la base de datos
5. Integración de módulos y validaciones
6. Pruebas unitarias e integración
7. Documentación y ayuda en línea
8. Despliegue en hosting y pruebas finales
9. Entrega y defensa del proyecto

## Recursos y logística
- Equipo de desarrollo (1-3 personas)
- Ordenador con Windows, Node.js, .NET 8, SQL Server
- Acceso a Azure o hosting web
- Herramientas: VS Code, Git, navegador web

## Permisos y autorizaciones
- Permisos de acceso a la base de datos y hosting
- Autorización para uso de imágenes y librerías externas

## Procedimientos de actuación
- Desarrollo iterativo y validación continua
- Uso de control de versiones (Git)
- Pruebas automáticas y manuales
- Revisión de código y documentación

## Identificación de riesgos y prevención
- Fallos en la base de datos: backups y pruebas de restauración
- Errores de validación: pruebas exhaustivas y mensajes claros
- Problemas de despliegue: documentación detallada y pruebas en local

## Asignación de recursos
- Tiempos estimados por fase (análisis, desarrollo, pruebas, documentación)
- Distribución de tareas según experiencia y disponibilidad

## Valoración económica
- Coste de licencias: 0€ (tecnologías libres)
- Coste de hosting: Azure gratuito o bajo coste
- Coste de recursos humanos: estimado según horas invertidas

## Documentación necesaria
- Manual de instalación y configuración
- Guía de uso y ayuda en línea
- Documentación técnica y de pruebas

## Gestión y registro de cambios
- Uso de Git para control de versiones
- Registro de incidencias y cambios en archivos de seguimiento
- Actualización de documentación tras cada cambio relevante
